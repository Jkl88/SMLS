# Техническое задание

## Система управления адресной светодиодной лентой WS2812B на базе ESP32

---

# 1. Общая концепция устройства

Устройство представляет собой интеллектуальную систему световой индикации движения на базе контроллеров ESP32. Система анализирует параметры движения объекта (скорость, ускорение, наклон и поворот) и автоматически включает соответствующие режимы светодиодной ленты.

Основные функции системы:

* анализ движения в пространстве
* определение скорости вращения
* определение ускорения
* определение поворотов
* ручное управление режимами
* отображение параметров на OLED дисплее
* отображение тех же параметров в WEB‑интерфейсе
* запись телеметрии во внутреннюю файловую систему LittleFS
* беспроводная передача команд управления

Система состоит из двух ESP32:

**ESP32 Master** — основной контроллер

Выполняет:

* сбор данных с датчиков
* вычисление параметров движения
* управление режимами
* управление дисплеем
* ведение логов в LittleFS
* WEB интерфейс
* передачу команд через ESP‑NOW

**ESP32 LED Controller** — исполнительный контроллер

Выполняет:

* приём команд по ESP‑NOW
* управление адресной лентой WS2812B

Связь между контроллерами осуществляется по протоколу **ESP‑NOW**.

# 3. Архитектура системы

Система состоит из двух логических узлов.

## ESP32 Master

Функции:

* сбор данных с датчиков
* фильтрация сигналов
* вычисление скорости
* вычисление ускорения
* вычисление углов наклона

# 4. Считывание данных с MPU6050

Используются два датчика MPU6050.

При старте устройства выполняется загрузка сохранённых калибровочных коэффициентов из Flash памяти ESP32.

При нажатии кнопки калибровки выполняется процедура калибровки.

Во время калибровки:

* мигает индикаторный светодиод
* на дисплее выводится

```
CALIBRATING
```

После завершения калибровки:

* офсеты сохраняются во Flash памяти
* офсеты выводятся в Serial

Используемые параметры:

* roll — угол наклона по оси X
* yaw — угол поворота по оси Z
* gyroX — угловая скорость по оси X
* gyroZ — угловая скорость по оси Z

Единицы измерения:

* углы — градусы
* угловая скорость — град/сек

---

# 5. Считывание данных с датчика Холла

Перед датчиком расположен вращающийся магнит.

Датчик фиксирует каждый оборот колеса.

Измеряется интервал между импульсами.

```
DT — время между импульсами
```

Диаметр окружности вращения

```
WHEEL_DIAMETER = 0.65
```

Длина окружности

```
CIRCUMFERENCE = WHEEL_DIAMETER * PI
```

Вычисляемые параметры:

LS — линейная скорость

```
LS = CIRCUMFERENCE / DT
```

ACCEL — линейное ускорение

```
ACCEL = (LS - SPEED) / DT
```

где

SPEED — скорость за предыдущий интервал

Также рассчитываются:

* CNT — количество оборотов
* Odom — суммарный пробег

Для управления лентой используются:

* LS
* ACCEL

Для устранения ложных импульсов применяется программная фильтрация.

---

# 6. Использование LittleFS для хранения данных

Для хранения данных используется внутренняя файловая система ESP32 — **LittleFS**.

В файловой системе сохраняются:

* файл логирования движения
* параметры калибровки
* настройки системы

Файл логирования:

```
/track.csv
```

В файл записываются:

* время
* LS
* ACCEL
* CNT
* режим ленты

При запуске устройства:

* проверяется наличие файловой системы
* при необходимости выполняется форматирование

Если файловая система готова:

на дисплей выводится

```
FS READY
```

Если произошла ошибка:

```
FS ERROR
```

---

# 7. Управление джойстиком KY‑023

Команды джойстика имеют приоритет над автоматическим управлением.

Доступные команды:

вперёд — GO

назад — STOP

влево — LEFT

вправо — RIGHT

вертикальное нажатие — SOS

Режим SOS активируется до повторного удержания джойстика более 200 мс.

Любая команда джойстика имеет приоритет над датчиками в течение 30 секунд.

После истечения времени управление возвращается к автоматическому режиму.

---

# 8. Логика определения режимов движения

```
if (LS <= 2.0)

HALT

else if (ACCEL < -0.5)

STOP

    если gyroX <= -2

    STOP_LEFT

    если gyroX >= 2

    STOP_RIGHT

else

GO

    если gyroX <= -2

    GO_LEFT

    если gyroX >= 2

    GO_RIGHT
```

# 9. Отображение информации на OLED

На дисплей выводятся следующие параметры:

* LS
* ACCEL
* CNT
* текущий режим

Если режим активирован джойстиком, отображается префикс:

```
Dj_GO
Dj_LEFT
Dj_RIGHT
Dj_STOP
Dj_SOS
```

---

# 10. WEB интерфейс

ESP32 Master должен предоставлять WEB интерфейс.

WEB интерфейс должен отображать ту же информацию, что и OLED дисплей:

* текущая скорость LS
* ускорение ACCEL
* количество оборотов CNT
* текущий режим ленты
* состояние файловой системы

WEB интерфейс должен позволять:

* запускать калибровку датчиков
* запускать TEST режим
* включать режимы GO / STOP / LEFT / RIGHT / SOS
* управлять записью логов

Все команды WEB интерфейса должны вызывать **те же функции**, которые используются при нажатии физических кнопок и джойстика.

WEB интерфейс не должен иметь отдельной логики управления — он должен использовать существующие функции системы.

---

# 11. Передача команд через ESP‑NOW

ESP32 Master передаёт команды ESP32 LED Controller.

Передаваемые команды:

```
GO
STOP
LEFT
RIGHT
GO_LEFT
GO_RIGHT
STOP_LEFT
STOP_RIGHT
HALT
SOS
TEST
```

Команда передаётся только при изменении режима.

# 12. Управление светодиодной лентой

Исполнительный контроллер управляет адресной лентой WS2812B.

Количество диодов

```
212
```

Реализуемые режимы:

* Right
* Left
* Go
* Stop
* Go‑Right
* Go‑Left
* Stop‑Right
* Stop‑Left
* Halt
* SOS
* TEST

---

# 13. TEST режим

После включения питания активируется режим TEST.

Все 212 диодов включаются и мигают.

Цвет:

```
RGB 202 82 242
```

Продолжительность режима

```
45 секунд
```

Если режим TEST не включился — связь между контроллерами отсутствует.

# 16. Результат работы системы

Система должна автоматически определять:

* движение
* торможение
* поворот

и включать соответствующие режимы светодиодной ленты, обеспечивая визуальную индикацию движения объекта.

* джойстиком
* кнопками
* WEB интерфейсом

при этом используется единая логика управления системы.
